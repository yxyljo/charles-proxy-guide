## Charles 是什么？

Charles 是一款 HTTP/HTTPS 网络代理与封包分析工具，由新西兰开发者 Karl von Randow（公司 XK72）开发维护。它的工作原理很简单：把自己变成你电脑的代理服务器，所有经过的网络请求都绕不开它，于是你就可以清清楚楚地看到每一条请求发了什么、服务器回了什么。

对前端开发者、后端工程师、移动端测试人员来说，这就是"透明眼"——接口出了问题，不用对着代码盲猜，直接把数据捞出来，一目了然。

## 核心功能，逐一说清楚

### 解密 HTTPS 流量

现在的接口基本都走 HTTPS，加密之后就看不到明文了。Charles 通过安装自己的根证书，充当中间人，把加密数据解成明文给你看。配置一次，后续 SSL 抓包就是常规操作了。

### 弱网模拟

产品经理总喜欢问"在 4G 环境下体验如何"，测试人员也需要验证弱网场景下 APP 会不会崩。Charles 内置网络限速工具（Throttle），可以模拟 56K Modem、3G、4G 各类网络环境，甚至可以自定义丢包率和延迟——不用去地铁里蹲信号了。

### 树状数据结构展示

接口返回的 JSON 或 XML 数据，Charles 会帮你格式化成树状结构，折叠展开随意操作，比对着一堆压缩文本找字段舒服太多。

### 断点拦截与修改

可以在请求发出前或响应返回前设置断点，实时修改参数或返回内容。用来测试边界值、模拟异常场景、调试特定逻辑，相当好使。

### 重复发送请求

调试时经常需要反复触发同一个请求，Charles 支持直接重发，还能在重发前修改参数——省去了反复操作前端页面的麻烦。

### AMF 格式支持

如果你还在维护涉及 Flash Remoting 的老项目（嗯，确实还有人在用），Charles 能解析 AMF 二进制格式，把消息内容可视化展示出来。
